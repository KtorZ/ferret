## Current State
- Repo: `konduit`
- Active frontend is the Ferret mobile wallet in `app/`, currently on local branch `ferret`.
- The app is no longer mock-kernel driven. `app/src/kernel.js` re-exports the generated WASM bindings from `app/src/wasm`, which is a symlink to `rust/crates/konduit-wasm/konduit-wasm-nightly-black-box`.
- Frontend architecture is now organized around focused model stores in `app/src/model/`:
  - `index.js` owns Konduit bootstrap, global readiness/busy state, auto-refresh, and top-level actions.
  - `wallet.js`, `channel.js`, `adaptor.js`, `connector.js`, `settings.js`, `fx.js`, and `ui.js` each own one slice of persisted state.
  - `store.js` provides localStorage-backed Svelte stores plus custom JSON serialization for WASM/domain types.
- `App.svelte` is now mostly shell/routing composition, rendering screens from `app/src/screens/` based on `app/src/routing.js`.
- Route model is explicit and flat: `/`, `/pay`, `/pay/confirm`, `/history`, `/topup`, and `/settings/*`.
- Launch and home share `/`: splash is shown while no wallet/runtime is available, and wallet home is shown once Konduit is initialized.
- Wallet creation/import now instantiates a real `Konduit` runtime with a generated or user-provided signing key persisted under `ferret.konduit.v1`.
- Runtime config is currently preprod-focused (`networkName: preprod`) with one predefined adaptor (`KtorZ` -> `https://konduit-server.fly.dev`) and one predefined connector (`api.ferret.channel` -> `https://api.ferret.channel`).
- Home screen is a two-card swipe deck:
  - L1 Cardano wallet card with balance, top-up, pending toggle, and currency toggle.
  - L2 Lightning/channel card with adaptor status, channel balance, top-up/add-funds, and currency toggle.
- The primary bottom CTA on home is dynamic: `Top-up`, `Open channel`, or `Pay`, depending on wallet funds and channel state.
- Activity/history pulls from both wallet and channel transactions; home shows the latest items and `/history` exposes the full feed.
- Channel model is singular, not plural: the frontend tracks one active channel with lifecycle steps like `OPENING`, `OPENED`, `CLOSING`, and `CLOSED`.
- Settings are split into dedicated sub-screens for adaptor, connector, network, language, currency, FX, wallet details, and wallet exit.
- Adaptor and connector settings support predefined or custom HTTP(S) URLs, with validation stored in `settings.js`.
- UI state keeps display currency mode (`ADA` vs configured fiat), selected wallet realm (`L1` vs `L2`), pending inclusion toggle, and a single global toast object.
- FX remains hard-coded for now (`USD`, `EUR`, `CHF`, `GBP`), and locale/network selectors are mostly placeholders around fixed runtime configuration.
- Auto-refresh runs every 20 seconds while the app is ready and there is pending wallet/channel work that may settle.
- Basic PWA support remains in place: manifest, icons, service worker registration, and connector-origin cache coordination.
- Root repo still contains the wider Konduit stack: docs, Aiken kernel code, and the Rust workspace with `konduit-wasm`, `konduit-server`, connector clients, and other supporting crates.
- Context file remains the root handoff document for new sessions.
